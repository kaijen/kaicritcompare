# Markup Types

kaicritcompare emits the same marker set that [kaicrit](https://github.com/kaijen/kaicrit) understands, so the two extensions work together: kaicritcompare produces the change markers, kaicrit reviews them.

| Change       | Syntax              | Meaning                                       |
| ------------ | ------------------- | --------------------------------------------- |
| Deletion     | `{--text--}`        | Text present in file 1, removed in file 2     |
| Addition     | `{++text++}`        | Text absent in file 1, added in file 2        |
| Substitution | `{~~old~>new~~}`    | `old` in file 1 replaced by `new` in file 2   |

Highlights (`{==…==}`) and comments (`{>>…<<}`) are part of CriticMarkup but are not generated by a diff.

## Substitutions

When `kaicritcompare.combineSubstitutions` is enabled (the default), an adjacent deletion and addition are merged into a single substitution marker:

```text
The {~~quick~>slow~~} brown {~~fox~>cat~~}
```

With the setting disabled, the same change is emitted as separate deletion and addition markers:

```text
The {--quick--}{++slow++} brown {--fox--}{++cat++}
```
